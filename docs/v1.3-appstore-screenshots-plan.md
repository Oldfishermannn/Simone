# Simone v1.3.0 — App Store 截图方案（draft）

> 给 CEO / UI Engineer 照着拍的 shot list。PM 只定内容和 caption，不负责真拍也不管 ASC 上传。
> 调性锚：**"AI mood radio. Press play."** · 温柔陪伴 · 都市夜晚 · 零引导自解释 · 克制。
> Caption 池：`ai mood radio` · `press play` · `drift` · `hours` · `tune in` · `swipe` · `no playlists` · `lo-fi chill` · `five stations`。

---

## 0. 总原则

- **全部 lowercase**，英文（品牌调：不喊，轻陈述）。
- **≤8 词/caption**，一句话为限。
- Caption 叠在截图上，顶部 1/6 安全区，不盖 visualizer 主体；字体沿用 Unbounded display / Fraunces italic accent（UI Engineer 判）。
- **不提商业化**（v1.3.0 没接 StoreKit，这版仍全免费——避免上架后用户找不到付费墙产生落差）。
- **不用** revolutionary / best / ultimate / world-class / 任何感叹号。
- 6 张叙事弧：**钩子 → 上手 → 丰度 → 呼吸 → 工具 → 回扣**。看完像听完一首 intro。

## 1. Shot List（6 张）

### 📸 1. Hero · Lo-fi Chill 初见（playing，full visualizer）

- **屏幕**：ImmersiveView，Channel = Lo-fi，Style = Lo-fi Chill（默认首屏），`isPlaying = true`，visualizer 大图（expansion = 1）展示 `LofiTapeView`（磁带绕线 + vinyl 暖 grain）。底部 transport row 三按钮可见（Settings · Play 暂停态·反——此刻是 Pause 图标，表示正在播；·Details）。styleName overlay "Lo-fi Chill" + DNA 行浮在下方。
- **Caption**：`ai mood radio. press play.`
- **为什么**：App Store 缩略图就靠这一张抓眼。复用 subtitle 一模一样，形成认知锚。对应新 feature：**Lo-fi 回首屏**（首启默认 Lo-fi Chill）+ **painted tape visualizer**（5 站各有其 visual weather 的代表）。一眼告诉用户：打开就是夜晚的 Lo-fi 房间。

---

### 📸 2. Radio Manual · how to tune in（OnboardingView，首启）

- **屏幕**：OnboardingView 展开状态——黑底 0.78 嵌套层盖住 Lo-fi 底图（tape visualizer 隐约可见，强化"annotation of live screen"而非独立教学页），白字纯排版，列 4 手势（← → 切频道 / ↑ ↓ 切 style / tap 播放暂停 / ⚙ settings / ☰ now playing），底部 "tap anywhere to begin" Fraunces italic。
- **Caption**：`four gestures. no menus.`
- **为什么**：第二屏直接把"怎么玩"一次性讲完，打消"又一个 AI 音乐 app 得学"的顾虑。对应新 feature：**OnboardingView（v1.3.0 新）** · Radio Manual 风格，零引导自解释的视觉证据。

---

### 📸 3. Five Stations · swipe sideways（切频道瞬间，visualizer 过渡）

- **屏幕**：ImmersiveView，TabView 横滑中间态——左侧 1/3 露 Lo-fi tape，右侧 2/3 已是 Jazz（OscilloscopeView 绿色波形线）。styleName overlay 正在从 "Lo-fi Chill" 切到 "Midnight Jazz" 的 in-between。底部频道指示（圆点 / 文字）暗示 5 站并排。
- **Caption**：`five stations. swipe to tune.`
- **为什么**：把最核心手势——**横滑切频道**（v1.2 就有，v1.3 沿用）——做成视觉叙事。Jazz 的 oscilloscope 绿色波形和 Lo-fi 的暖调磁带对冲，顺带秀 **painted visualizer per station** 的丰度。注意：截图拍"过渡中"的一帧比静止某一站更有"radio dial 转动"的意味。

---

### 📸 4. Evolve · drift for hours（playing 久图，DNA 行呼吸）

- **屏幕**：ImmersiveView，Channel = R&B，Style = Neo Soul（或任一 R&B style），`isPlaying = true`，**LiquorView**（频谱威士忌液面起伏）满屏。styleName overlay 下方 DNA 行显示三维度指示：`instruments +1 · density steady · energy breathing`（或当前 UI 实际渲染的漂移 meta；若无具体文字，至少保留 DNA 行可见）。
- **Caption**：`evolve drifts across instruments, density, energy.`（≤8 词 check：`evolve / drifts / across / instruments / density / energy` 6 词 + 逗号分隔，OK）。若嫌长备选：`quiet drift. not jump cuts.`
- **为什么**：v1.3.0 Promotional Text 的原话就是 "Evolve got deeper. Music drifts across instruments, density, and energy — Simone stays fresh for hours." 这一屏必须 echo。对应新 feature：**Evolve 深化三维度**（v1.2.1 底子，v1.3.0 继续）+ **R&B painted visualizer**（liquor decanter）。色温从 Lo-fi 暖 / Jazz 冷绿切到 R&B 茄紫，视觉节奏变化一次。

---

### 📸 5. Details · what's on now（paused 缩图，DetailsView 或底部信息）

- **屏幕**：ImmersiveView，`isPlaying = false`，visualizer 缩成小圆（expansion = 0），底部 transport row Play 按钮 highlight。**按右下 ☰ 打开 DetailsView sheet**（native sheet，显示当前 channel / style 名、sub-style 列表、instrument pool、Evolve pace 挡位 10s/1m/5m/Lock）。或如果 DetailsView sheet 不便截，退而求其次：拍 paused 缩图态（visualizer 退回小图，底部 play 亮起，styleName overlay 仍在）。
- **Caption**：`tap pause. it shrinks. tap play. it blooms.`（8 词 check ✓）。简版备选：`pause shrinks. play blooms.`（4 词）
- **为什么**：秀 **play/pause 缩图 morph** 这个 v1.2.1 新交互（What's new 列的第 3 点）+ **DetailsView sheet**（v1.3.0 新接的 native sheet）作为"不想调也能看清此刻在播什么"的安心感。这张是"工具感"那一格——告诉用户"你能看清，且可以进一步挖"。

---

### 📸 6. Settings · one calm screen（SettingsView sheet）

- **屏幕**：SettingsView（v1.2 改造过的单屏 Settings，native sheet 从底滑出）。关键信息可见：API Key 段（Built-in Trial / Bring Your Own）· Evolve pace 挡位 · Sleep timer · Visualizer 选择（默认 / 自定义）。背景仍是主图半透，保持"嵌在电台里"的感觉。
- **Caption**：`your key. your pace. your radio.`（6 词 ✓）。备选：`one screen. no hunting.`（v1.2 What's New 原句精简）
- **为什么**：给重度用户 / 评审 / 隐私敏感用户看见 **BYOK + Keychain** 的安心感（"Your key lives in the iOS Keychain"）。对应产品定位里的"零引导自解释"最终证据——**所有控制都在一屏**。收尾把叙事从"电台氛围"拉回"工具可控"，尾劲。

## 2. 拍摄步骤（CEO / UI Engineer 用 iPhone 17 Pro Max 模拟器一步步复现）

> 所有步骤基于 `simone ios/Simone.xcodeproj`，target device = **iPhone 17 Pro Max** (iOS 18+)，深色模式默认，`hasSeenOnboarding_v1_3` 先清（首次启动看 Onboarding）。
> 拍照快捷键：模拟器 ⌘S（save screenshot to Desktop）或 Device → Screenshot。
> 单张建议先"进入目标状态 → 等 2 秒 visualizer 稳定 → ⌘S"。

### 准备

1. 打开 Xcode：`open "/Users/oldfisherman/Desktop/simone/simone ios/Simone.xcodeproj"`
2. Scheme 选 Simone，Device 选 iPhone 17 Pro Max。Cmd+R 跑起来。
3. 如已看过 Onboarding：在模拟器里 Device → Erase All Content and Settings，或者 app 内手动改 UserDefaults（重新装 app 更快）。
4. **确保已接入内置 Key 或 BYOK**（Review Notes 走内置 Key 即可，截图也用内置 Key，避免 BYOK 字段暴露）。
5. **先听 30 秒**确认有声 + visualizer 在动（HMR 不靠谱，改完必重启整段听——CLAUDE.md 铁律）。

### Shot 1 — Hero · Lo-fi Chill

1. 冷启 app（首次看完 Onboarding 后继续）。默认 Channel = Lo-fi，Style = Lo-fi Chill。
2. Tap 中央 visualizer 或底部 Play 按钮 → 开始播放。
3. 等 3 秒 LofiTapeView 磁带卷到中段、grain 自然、styleName "Lo-fi Chill" overlay 已淡入。
4. ⌘S。

### Shot 2 — Radio Manual · Onboarding

1. Erase App（或新装）→ 冷启。
2. OnboardingView 会自动覆盖在 Lo-fi 底图上（黑底 0.78 + 白字纯排版）。
3. 不要 tap（一 tap 就退）。等 0.5 秒 opacity 淡入完成。
4. ⌘S。

### Shot 3 — Swipe to Tune（横滑切频道瞬间）

1. 从 Shot 1 的状态（Lo-fi 播放中）出发。
2. 从右往左缓慢拖动（模拟器按住鼠标左键拖），拖到约 40% 页宽（Lo-fi 1/3 在左 · Jazz 2/3 在右）停住不松手。
3. ⌘S（保留拖拽中间态）。
4. **备选**：若中间态难稳定，直接切到 Jazz 站稳态 + "Midnight Jazz" styleName overlay 显示，也能用，只是叙事不如中间态。

### Shot 4 — Evolve Drift（R&B + Liquor）

1. 从任意状态出发，横滑到 R&B 站。
2. 纵滑选 Style = Neo Soul（或任一 R&B style，Liquor visualizer 稳）。
3. Tap 播放，等 5-10 秒 LiquorView 液面有明显起伏、频谱对应瞬时能量变化。
4. 若 DNA 行显示 instruments/density/energy meta → 等该行淡入后 ⌘S。
5. 若 DNA 行只是装饰文字 → 随便一帧 ⌘S 即可，caption 承载语义。

### Shot 5 — Pause Shrinks / Details Sheet

**Option A（推荐，展示 paused 缩图 morph）**：

1. 从 Shot 1 继续（Lo-fi 播放中）。
2. Tap visualizer 或 Play 按钮暂停。
3. 等 0.55 秒 morph 完成（visualizer 缩成小图）。
4. ⌘S（抓住三按钮 row + 小圆图状态）。

**Option B（如果要秀 DetailsView sheet）**：

1. 从 Shot 1 继续（播放或暂停都行）。
2. Tap 右下 ☰（Details 按钮）。
3. 等 sheet 滑上来稳定。
4. ⌘S。

建议 A（更独特、更能讲 v1.2.1/v1.3 morph 交互）。

### Shot 6 — Settings · One Calm Screen

1. 任何状态下 tap 左下 ⚙（Settings 按钮）。
2. 等 sheet 滑上来稳定。
3. ⌘S。
4. **注意**：若 BYOK 字段已填过 key，先 Erase App 或手动清，避免真实 key 暴露（即便是内置 Key placeholder，也确认显示的是 "Built-in Trial" 而非个人 key 残影）。

## 3. 排序逻辑

| # | 主题 | 叙事位 | 情绪 |
|---|---|---|---|
| 1 | Hero · Lo-fi Chill | 钩子 | 温柔 |
| 2 | Onboarding · 四手势 | 上手 | 安心 |
| 3 | 横滑切 5 站 | 丰度 | 张力 |
| 4 | Evolve 三维度漂移 | 呼吸 | 深度 |
| 5 | Pause 缩 / Play 绽 | 工具 | 反馈感 |
| 6 | Settings 一屏 | 回扣 | 克制 |

**为什么这个顺序**：

- **第 1 张必须是"press play 的画面"**——App Store 搜索结果页只看得到前 2-3 张缩略图，必须在第一张就讲完"这是 AI mood radio 点了就响"。
- **第 2 张消除认知成本**——用户刚被第 1 张抓住，下一秒想的是"我得学啥"，Onboarding 直接告诉他"就 4 个手势"。
- **第 3 张拉广度**——证明不是"一个 Lo-fi 播放器"，而是 5 站电台。
- **第 4 张拉深度**——证明"单站内也不腻"，对应 Promotional "stays fresh for hours"。
- **第 5 张给信任感**——展现"app 对我的操作有反应"的细节设计（paused 缩图是 v1.3 ship list 里的亮点）。
- **第 6 张收束克制**——Settings 一屏、BYOK 可选，暗示"你有掌控权但不需要配置"。首尾都是"calm"。

### Caption 关键词池（优先复用，别另起炉灶）

| 词/短语 | 出处 | 用法 |
|---|---|---|
| `ai mood radio` | subtitle | 第 1 张必出 |
| `press play` | subtitle | 第 1 张必出 |
| `drift` / `drifts` | v1.3 Promotional | 第 4 张（Evolve 专用） |
| `hours` / `for hours` | Promotional | 第 4 张可选结尾 |
| `tune in` / `tune to` | Description 开篇 | 第 3 张可选 |
| `swipe` | onboarding + description | 第 3 张 |
| `no playlists` / `no menus` | 反定位句 | 第 2 张 |
| `lo-fi chill` | 默认 style 名 | 第 1 张副文 |
| `five stations` | Description | 第 3 张核心词 |
| `quiet` / `calm` | Description 通用 | 第 5/6 张 |

### 严格禁用

- `revolutionary` · `best` · `ultimate` · `world-class` · `game-changing` · `AI-powered` 作形容词（AI 是背景不是卖点）
- 感叹号 `!`（全篇 zero tolerance）
- Title Case（除品牌名 Simone / 频道名 Lo-fi/Jazz/R&B/Rock/Electronic/Ambient）
- 引号 / 表情符 / emoji
- "subscribe" / "premium" / "unlock" / 任何商业化暗示（v1.3.0 未接 StoreKit）
