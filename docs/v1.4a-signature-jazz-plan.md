## 1. 从 Lo-fi 学到的（战略师复盘）

### 1.1 draft 配方 vs 最终落地

| 维度 | draft §4.2 原配方 | dcef825 最终 ship | 偏离原因 |
|---|---|---|---|
| 图腾 | 磁带卷轴（两圆 + 胶带） | **deck（录音机本体）** | CEO 审时觉得磁带卷轴"太小清新"，不够设备感 |
| 频谱形态 | oscope 单线被胶带束缚 | **VU LED ridge 频谱**（嵌入 deck 屏幕） | oscope 波形信息量不够"硬件感"，VU ridge 有专业设备气质 |
| 材质 | 颗粒粉尘 | 颗粒保留，但被 framed 低对比压下去 | CEO 要"材质不抢戏" |
| 颜色 | 玉粉黛 `#e8c8be` | 玉粉黛 + cassette 棕灰调色（ridge 融入家族） | ✅ 守住 |

### 1.2 tacit insight

- **CEO 偏好"设备 + 屏幕"而不是"实物 + 波形"**：设备有"在运转"的在场感，屏幕式频谱有"专业仪器"的可信感
- **减法比加法重要**：6 次迭代里 4 次是"藏掉 LED/counter/buttons"、"mute ridges"、"lowered contrast"——每次都是减法
- **材质是气氛不是主角**：draft §3.2 写了但 Lo-fi 落地证明 CEO 真的会因为颗粒抢戏而要求压下去

### 1.3 应用到 Jazz

Jazz 配方建议**同系列继承**：设备型图腾 + 屏幕式频谱 + 沙金主色 + 烟雾作氛围（烟雾 < 15% 画面）。不守 draft §4.2 的"黑胶俯视 + 12 radial 线"——那是实物+波形路子，已经被 Lo-fi 实证否决。

## 3. Jazz 技术路径（战略建议，非代码规范）

### 3.1 视觉规格（🔴 待 CEO 拍路径 A/B 后定稿）

#### 路径 A · 守 draft §4.2（黑胶俯视 + 12 radial）

- **构图**：黑胶唱片俯视居中，中心标签（Simone logo 替代）+ 12 根放射线从标签向外发散到唱片边缘。唱针从右上 -45° 斜插入
- **频谱驱动**：每根 radial 线长度 = 对应频段能量（12 频段采样），唱针随 low 频微微颤动
- **材质**：唱片外围淡烟雾渐变（radial-gradient ellipse）
- **颜色**：沙金 `#c9b287` 主色 + 黑胶纯黑
- **叙事**：地下酒吧老派 Jazz 场，烟弥漫

#### 路径 B · 继承 Lo-fi deck 系列（turntable + 屏幕式频谱）**（推荐）**

- **构图**：**黑胶转盘机侧 3/4 视角**（不是俯视）——底座 + 旋转平台（黑胶 disc 在上）+ 唱臂斜置右侧 + 左侧操控面板（一块小屏 + 2-3 个旋钮/推杆）
- **频谱位置**：**屏幕式 VU meter 在左操控面板**（不在黑胶上），同 Lo-fi ridge 族但**水平式布局**（Jazz 是横摆 meter，Lo-fi 是竖摆 VU → 形成"同族不同摆位"的系列感）
- **频谱驱动**：ridge-style bars（12-16 根）响应 FFT，Evolve 触发时唱臂轻抬 0.5° + VU 背光脉冲一次
- **材质**：烟雾 `SmokeMaterial` 从左上角渗入，**总占画面 < 15%**（守 Lo-fi 经验）；黑胶 disc 表面反光（线性 gradient sweep 慢速）
- **颜色**：沙金 `#c9b287` 主色（deck 金属外壳）+ 黑胶纯黑 + VU 屏背光暖白（和 Lo-fi 屏调色同族）
- **叙事**：酒吧打烊后 DJ 独自试听的 turntable，烟从烟灰缸飘过来

### 3.2 工程建议（iOS Engineer 判断最终实现）

- **FFT 订阅**：复用 Lo-fi 已打通的 `AudioEngine` FFT 路径，VU ridge 组件抽出来作 `VUMeter.swift`（Lo-fi + Jazz 共享，参数化"纵/横、bar 数、色调"）——但 🟠 抽象代价看 iOS Engineer 判断，如果 2 文件分别维护更简单就别强抽
- **黑胶 + 唱臂**：SwiftUI Shape/Path + 旋转 Animation（慢速 1 rpm · 33.3 rpm 不需要因为视觉上 33 rpm 肉眼看得到转但太快会抢戏——观点值，最终 CEO 审时再调）
- **烟雾**：`SmokeMaterial` = radial-gradient ellipse + blur + TimelineView 随机位置抖动 + opacity < 0.4
- **屏幕 VU**：如果沿用 Lo-fi 的 `LofiSignatureView` 里 VU ridge 组件，考虑抽 `VUMeterView` 共享；否则 Jazz 独立实现也 OK（Lo-fi 也没强求抽象）

### 3.3 红线

- **频谱是主角不是装饰**：关掉音乐，VU 屏应该"死掉"（黑胶继续慢转、烟继续飘 OK，但屏幕上频谱必须停）——肉眼可辨
- **画面占比**：turntable 设备 < 65% 画面 · VU 屏 < 15% · 烟雾 < 15% · 背景 ≥ 5%
- **性能**：iPhone 15 Pro 稳 60 FPS，iPhone 12 稳 30 FPS——黑胶转和烟雾是连续动画，要注意 CPU 预算
- **不抢 Lo-fi**：Jazz 的设备金属+沙金必须和 Lo-fi 的 cassette 棕灰**明显区分**（一眼分辨两个频道不一样），但**审美同族**（设备系列感）

## 6. 分支 & PR 策略（守 CEO 拍板的 B 路径）

### 6.1 分支

- **同分支继续叠**：iOS 子仓 `feature/v1.3+signature-lofi`（当前 HEAD `dcef825`）
- **不开新分支**（除非 CEO 觉得需要隔离审 Jazz——那就 branch out `feature/v1.3+signature-jazz` off `dcef825`，但不推荐，管理负担）
- **建议 rename**（事后可做）：CEO 审过 Jazz 后 `git branch -m feature/v1.3+signature-lofi feature/v1.3+signatures`（复数）反映集成意图，但不紧急

### 6.2 v1.3 并行冲突风险

- Jazz 改的文件：`Views/Signatures/JazzSignatureView.swift`（新增）· `Views/Materials/SmokeMaterial.swift`（新增）· `SpectrumCarouselView.swift`（少量 flag 分支）· `ImmersiveView` 同 Lo-fi 扩展 isSignatureJazz
- v1.3 改的文件：`StoreKitManager.swift` · `UsageLimiter.swift` · `PromptBuilder.swift` · `ChannelTier.swift`
- **冲突面几乎为零**（Jazz 只动 View 层，v1.3 只动 Model/Manager 层）
- **最终 merge 策略**：等 v1.3 StoreKit 完工 + 5 个 Signature 频道都审过 → 一次性大 merge `feature/v1.3+signatures` → main，TestFlight 一次上

### 6.3 回滚

- Jazz 出问题 → `git revert` Jazz 的 4-5 个 commits（M1-M4），Lo-fi + v1.3 其他部分不受影响
- Settings flag 默认 Signature：如果 Jazz 模式崩，**hotfix 默认回 Classic** 即可临时兜底
