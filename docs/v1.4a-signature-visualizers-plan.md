## 2. Lo-fi Signature 技术路径（战略建议，不是代码规范）

### 2.1 视觉规格（摘自 draft §4.2）

- **构图**：画面中央水平磁带卷轴（左右两圆 + 连接胶带）。胶带中间一根 oscope 时域波形线，**被胶带束缚不越出**。
- **频谱驱动**：oscope 波形线实时由 FFT 驱动——low 频影响粗细、mid 频驱动振幅、high 频驱动细碎锐齿。
- **材质层**：背景散布颗粒粉尘（<15 颗），缓慢下沉。
- **颜色身份**：玉粉黛 `#e8c8be`（主色）+ 磁带棕灰（卷轴）。
- **叙事**：磁带回放夜晚的私人录音。

### 2.2 工程建议（iOS Engineer 判断最终实现）

- **音频订阅**：复用现有 `OscilloscopeVisualizer` 的 audio buffer 订阅路径，不新写 FFT。Lo-fi oscope 是时域波形（不需要频段分离），现有能力够用。
- **磁带容器**：SwiftUI Canvas / SVG path 画卷轴（两圆 + 胶带矩形）。胶带宽度决定波形线振幅上限（"束缚"的视觉翻译）。
- **颗粒粉尘**：`TimelineView` + 少量 SpriteKit/Canvas 粒子（<15 颗，性能零风险）。
- **无图腾帧**：CEO 拍板时如果说"静帧不够磁带感"——回炉加磁带物理细节（卷轴孔、胶带转动惯性）。

### 2.3 红线

- **频谱是主角不是装饰**：关掉音乐，画面应该明显"死掉"（静帧卷轴 + 死线）。Lo-fi 波形线必须真响应 audio buffer，非开环动画。
- **画面占比**：磁带卷轴 < 60% 画面 · oscope 波形线在胶带中 · 颗粒 < 15 颗。材质不抢戏。
- **性能**：iPhone 12（最低支持）稳定 30 FPS 以上。如果掉帧先砍颗粒数。

## 5. 分支 & PR 策略

### 5.1 分支

- `feature/v1.4a-signature-lofi` off `main`（**不 off `feature/v1.3-monetization`**，避免耦合）
- 完工开 PR → main
- CEO 审过 + PR 合并后，再 branch out `feature/v1.4a-signature-jazz` 做第二个

### 5.2 v1.3 并行冲突风险

- Lo-fi 改的文件：`Views/Signatures/*`（新增）· `Views/Materials/*`（新增）· `Views/SpectrumCarouselView.swift`（flag 分支，少量改）· `Models/AppState.swift`（加 flag 字段）
- v1.3 改的文件：`StoreKitManager.swift` · `UsageLimiter.swift` · `PromptBuilder.swift` · `ChannelTier.swift`
- **冲突面最多在 `AppState.swift`**（两个分支都可能加字段）——merge 时手工调，风险小
- **策略**：v1.4a-signature-lofi 合并优先级 = CEO 审过后立刻合，让 v1.3 分支 rebase 时只合一次

### 5.3 回滚

- Lo-fi PR 合并后发现问题 → `git revert -m 1 <merge_commit>`，一键回 Classic
- Settings flag 默认 Signature，老用户第一次打开切 Signature；如果 Signature 模式出问题，**hotfix 改默认回 Classic** 即可临时兜底
