# Simone 运作原则

> CEO 老鱼时间最贵。公司所有流程围绕一件事：**让 CEO 的待决策数量趋近于零**。

## 决策权限（0-决策原则）

### 🔴 必须回 CEO 拍板（三类，其余全员工自决）

1. **不可逆的钱** — 订阅定价 / 绑定新云服务 / 购买付费 SDK / 合同类支出
2. **对外发布** — App Store 提交 / TestFlight 外部邀请 / 社交媒体发帖 / 官网改动
3. **产品定位** — 品牌名 / tagline / 核心人格 / 版本主题切换

### 🟢 员工全权自决（事后 daily 报备即可）

- 技术栈选型、代码架构、第三方库升级
- UI 微调、文案润色、动效参数
- 员工之间任务调度、角色招募（COO 权限）
- 分支策略、commit 粒度、回滚决定
- Bug 排查路径、测试方案

### ⚠️ 汇报方式

- **不要堆 A/B/C 选项给 CEO**。选好最优解 + 备好一条 fallback，只在 🔴 三类时告诉他。
- 其他改动写进当日 `docs/daily/` 即可，CEO 早上看。

## 员工交接规则

### 跨员工协作

1. 交付物放 `docs/`，文件名带角色和阶段（例：`v1.2-appstore-copy.md` 是 PM 的）
2. memory 系统所有窗口共享，是全公司长期记忆
3. `docs/team-status.md` 每人一行实时状态，**覆盖式更新**不 append
4. 状态字段：🟢 IDLE / 🟡 WORKING / 🔴 BLOCKED / ✅ DONE

### 不越权

- 不要一个员工抢另一个员工的活
- 发现跨界需求 → 通过 COO 调度，或在 status 板留言 `需要 XX 接手`
- COO 统一调度，其他角色横向不直接互相指派

### 卡住时

- 🔴 BLOCKED 立即更新 status 板，注明等谁
- COO 主动扫 status，帮接、帮转、帮升级

## 日报机制

COO 每晚写 `docs/daily/YYYY-MM-DD.md`（模板见 `docs/daily/TEMPLATE.md`），固定三段：

1. **今天动了啥** — 按角色列 bullet（谁交付了什么）
2. **明天谁在跑** — 明天预定任务 + 谁负责
3. **待 CEO 拍（🔴 only）** — 三类决策，带选好的方案 + 一句理由

CEO 次日早上只看这一份，不翻 commits、不翻 status。

## 可逆原则（继承 CLAUDE.md）

所有操作必须可逆：git 粒度细、不做破坏性删除、保留 feature flag、数据 schema 迁移双写一版。
