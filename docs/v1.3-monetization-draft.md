## 0. 设计立场（先讲清楚）

Simone 不是 Spotify、不是 Suno，是 **AI Ambient Radio**。商业化的最高约束有两条，违反任何一条直接推倒重来：

1. **不破坏沉浸感**——付费墙不能在用户正在聆听时打断，不能盖住频谱，不能弹出一个全屏遮罩让用户停下来读字
2. **不破坏"零引导"原则**——付费墙本身也要自解释，没有教程式 banner、没有"了解订阅"的二级跳转

所以本草稿在三个地方做了和"标准 SaaS 付费墙"不一样的处理：付费墙触发的时机、文案密度、Free 档的功能边界。

## 1. 定价策略

### 1.1 三档功能边界（基于 SimonePlan Phase 2.6 + v1.3 章节，未发明任何新功能）

| 能力 | Flow（免费） | Tune | Studio |
|---|---|---|---|
| **播放时长** | 不限 | 不限 | 不限 |
| **调台方式** | ◁▷ **当前频道内**循环换风格 | + 点击具体风格直接切 | + Direct Input（自由文本 prompt） |
| **频道页** | 可见、可进入、可在内部 ◁▷ | + 点列表里任意一个直接切 | 同 Tune |
| **收藏（Favorites Tab）** | 不可（点 ♡ 弹升级） | ✅ | ✅ |
| **Evolve** | Lock / 10s / 1m / 5m 全档可选 | 同 Flow | + Evolve 深度参数（v2.0 落地后开放） |
| **Sleep Timer** | 全档可选 | 同 Flow | 同 Flow |
| **Visualizer** | 5 频道默认 visualizer（tape/oscilloscope/liquor/ember/matrix，频道绑定，不可改） | 同 Flow | + 自定义 visualizer（v2.2+ 落地后开放） |
| **拼标签生成新风格** | 不可 | ✅（Genre + Mood + Instrument） | ✅ |
| **BPM / Temperature 高级参数** | 不可 | 不可 | ✅ |
| **多风格混合 / 离线 / 导出** | 不可 | 不可 | v2.2+ 解锁，v1.3 不交付 |

**关键 PM 判断**：
- **Flow 给"完整电台"体验**（播放不限时、Evolve 全档、Sleep 全档、5 个默认 visualizer 随频道切换），**只锁住"主动选择"和"持有感"**（点风格 / 收藏 / 创建）。理由：氛围电台的核心价值是"打开就有声"，不是"我可以选什么"。如果 Flow 把播放时长砍掉、Evolve 砍掉，那 Simone 的产品定位就被破坏了，新用户在 24h 留存还没建立前就被付费墙劝退
- **Tune 是"我开始有自己的电台"的临界点**——加收藏、加点选、加拼标签。这是绝大多数付费用户应该停的地方
- **Studio 是给"想把 Simone 当乐器的人"**——Direct Input + BPM/Temp。这部分人少但 ARPU 高，价格定高一点不亏

### 1.2 定价数字（推荐方案）

| 档位 | 月订阅 | 年订阅 | 买断 |
|---|---|---|---|
| **Flow** | Free | Free | Free |
| **Tune** | **$1.99 / 月** · ¥12 | **$14.99 / 年**（省 $9，约 6.3 折）· ¥88 | **$9.99 一次性** · ¥58 |
| **Studio** | **$4.99 / 月** · ¥30 | **$39.99 / 年**（省 $20，约 6.7 折）· ¥218 | **$19.99 一次性** · ¥118 |

**关于年订阅（Plan 文档没明写，是战略师补的推荐）**：
- 月订阅是入门钩子，年订阅是主推（年订阅 LTV 是月订阅的 2-3x，且续订率高）
- 年订阅折扣给到 ~37% off（比月付 12 个月便宜 ~$9 / $20），属于行业标准甜点区间
- App Store 业内常态是把年订阅做主默认选项，月订阅次之，买断单独一行

**关于买断（沿用 SimonePlan 已有规划）**：
- Tune 买断 $9.99 = 5 个月月费
- Studio 买断 $19.99 = 4 个月月费
- 买断对 Simone 这种"陪伴型工具"是真正的差异化卖点——大部分用户讨厌订阅，"买断了就是我的"是非常强的转化语言
- **Tune 买断 → Studio 升级补差价 $10**（沿用 Phase 2.5 设计）

**人民币定价用 App Store 中国区档位四舍五入**（Apple 有固定档：¥6/¥12/¥18/¥30/¥45/¥58/¥88/¥118/¥218），上面数字都已落到合规档位。

### 1.3 前 100 名 50% off 的实施方式

**推荐方案：StoreKit 2 Introductory Offer + 后台 server-side 计数门槛**——但 Simone 是零服务器架构，所以做**简化版**：

**方案 A（推荐，零服务器）**：用 **App Store Connect 的"促销价（Promotional Offer）+ 时间窗口"**，不限 100 名，限"v1.3 上架后前 14 天内**首次订阅** Tune/Studio 年订阅"享 50% off。

- **理由**：
  1. "前 100 名"在零服务器架构下无法精确计数（StoreKit 不暴露全球计数），强行做需要起后端
  2. "前 14 天 + 首次订阅"在用户感知上等价于"早鸟红利"，而且实施只是 App Store Connect 配置一个 Introductory Offer
  3. 时间窗口是天然 FOMO，比"前 100 名"更直接（用户不用猜还剩几个名额）
- **配置**：
  - Tune Yearly：Introductory Offer = "Pay As You Go" → 第一年 $7.49（50% off $14.99），续订恢复全价
  - Studio Yearly：Introductory Offer = "Pay As You Go" → 第一年 $19.99（50% off $39.99），续订恢复全价
  - Eligibility = "New Subscribers"（仅新订阅用户，不能让已订阅用户重复薅）
  - 时间窗口 = v1.3 上架日 + 14 天（App Store Connect 后台手动开关）

**方案 B（备选，如老鱼坚持"前 100 名"叙事）**：用 **Promo Code**——App Store Connect 每个 App 每季度可生成 1000 个 Promo Code，老鱼手动发前 100 名（社群、TestFlight、邮件）50% off 的兑换码。
- 缺点：手动运营成本、用户得知"有码"才能用、不在 App 内自动呈现
- 优点：精确控制 100 名

**战略师推荐**：A 方案。理由是 Simone 没社群也没邮件列表，"发码"渠道几乎不存在；A 方案用时间窗口替代名额限制，对零服务器架构友好，App 内 UI 直接显示"早鸟价"标签即可。

## 2. 付费墙触发逻辑

### 2.1 何时弹（必须遵守"不破坏沉浸感"）

**触发点（且仅在以下时机）**：
1. 用户在 Channels 页**点击具体风格名**（不是横滑）→ 弹升级（Flow 不能直选）
2. 用户在 Channels 页**点 ♡ 收藏按钮** → 弹升级
3. 用户在 Channels 页**点"+ New Style"** → 弹升级
4. 用户在 Settings 页**点 BPM / Temperature 滑块**（Studio 锁定项） → 弹升级
5. 用户在 Settings 页**点"Direct Input"入口** → 弹升级（仅 Studio）

**严禁触发点**：
- ❌ 启动 App 时（启动直接进 Lo-fi Chill 默认频道，不弹任何东西）
- ❌ 播放过程中任何时刻（即便用户什么都没操作）
- ❌ 横滑切频道时（横滑是 Flow 都能做的事）
- ❌ Sleep Timer 倒计时结束时
- ❌ Evolve 触发时
- ❌ 不出现 banner / toast / push notification 形式的付费推广

### 2.2 付费墙 UI 调性（延续 Fog 主题）

**形态**：bottom sheet（半屏抽屉），非全屏遮罩。背景保持频谱继续渲染，**音乐继续播放**——这是"不破坏沉浸感"的关键实现细节。

**布局（从上到下）**：
- 顶部小 grabber（可下滑关闭）
- **标题**：用 Unbounded Light 字体，"Tune your station." / "Shape your atmosphere."（对应 Tune / Studio）
- **Fraunces 斜体副标**：一句话说明这一档的核心价值（"Pick exactly the mood you want." / "Direct prompt. Full control."）
- **价格主区块**：
  - 大按钮 = **年订阅 $14.99**（标"Save 37%"或"Early Bird 50% off"）
  - 中按钮 = **买断 $9.99**（标"One-time purchase"）
  - 小字 link = "Or $1.99/month"（月订阅折叠为文本链接）
- **小字**：Restore Purchases · Terms · Privacy
- **底部**："Maybe later"（关闭按钮，**不打勾、不强制选中、不变色**——用户能轻松关掉）

**配色**：Fog 冷色调，价格主按钮用 mauve 强调色（Fog v5 的"贵重色"），其他全灰阶。**不出现红色降价标签、不出现倒计时、不出现"仅剩 X 天"等焦虑触发器**——Simone 的人格是"温柔陪伴"，不是 Temu。

**早鸟标签**：v1.3 上架后 14 天内显示一个静态小标签 "Early Bird · 50% off first year"，挨着年订阅按钮。14 天后自动消失，不弹通知不提醒。

### 2.3 BYOK 用户和付费用户的关系

**这是关键政策决定，老鱼必须明确接受或否决：**

**推荐方案**：**BYOK 用户 ≠ 免费 Studio。BYOK 和付费分层是两条独立维度。**

- **BYOK 解决的是"API 配额 / 速率"**——用户填自己的 Gemini Key，无限调用、不受 Simone 内置 Key 配额限制
- **Flow / Tune / Studio 解决的是"产品功能边界"**——用户能不能选风格、能不能收藏、能不能 Direct Input
- **两者交叉矩阵**：

| | 内置 Key | BYOK |
|---|---|---|
| **Flow** | ✅ 受内置 Key 配额（v1.3 不再设 30 分钟限制，但内置 Key 有 Google 侧速率限制） | ✅ 无限调用 |
| **Tune** | ✅ 受内置 Key 配额 | ✅ 无限调用 |
| **Studio** | ✅ 受内置 Key 配额 | ✅ 无限调用 |

**理由**：
1. BYOK 用户付的是 Google 的钱（API 用量），不是 Simone 的钱。Simone 的产品价值（精选频道、Fog 视觉、Evolve 算法、可视化器）和"用谁的 Key"无关
2. 如果 BYOK 自动等于 Studio，那本来愿意付 $4.99/月的"轻度技术用户"会全部走 BYOK 套利，伤 ARPU
3. 但**应该给 BYOK 用户一个小特权**：BYOK 用户在升级弹窗里看到一个**额外按钮 "I'm a BYOK user · Unlock Tune for $4.99 buyout"**——专属买断价格优惠 50%，作为"你已经懂技术 + 自带 Key 的折扣"。这是温和的差异化，不破坏分层逻辑

**风险**：如果老鱼觉得这对 BYOK 用户"不够厚道"，备选方案是 **BYOK 自动解锁 Tune（不解锁 Studio）**——但战略师不推荐，因为 Tune 是主要付费档，让出去会很痛。

## 3. StoreKit 2 集成方案

### 3.1 SKU 结构

**Subscription Group: `simone.subscriptions.main`**（一个 group，强制互斥升降级）

| Product ID | 类型 | 显示名 | 价格 |
|---|---|---|---|
| `com.simone.ios.tune.monthly` | Auto-Renewable Subscription | Tune Monthly | $1.99 / month |
| `com.simone.ios.tune.yearly` | Auto-Renewable Subscription | Tune Yearly | $14.99 / year |
| `com.simone.ios.studio.monthly` | Auto-Renewable Subscription | Studio Monthly | $4.99 / month |
| `com.simone.ios.studio.yearly` | Auto-Renewable Subscription | Studio Yearly | $39.99 / year |

**Non-Consumable（买断，独立于 group）**：
| Product ID | 显示名 | 价格 |
|---|---|---|
| `com.simone.ios.tune.lifetime` | Tune Lifetime | $9.99 |
| `com.simone.ios.studio.lifetime` | Studio Lifetime | $19.99 |
| `com.simone.ios.studio.upgrade` | Tune → Studio Upgrade | $10.00（仅当用户已有 tune.lifetime 时可购买） |

### 3.2 Subscription Group 设计

- 4 个订阅 SKU 全部放在 `simone.subscriptions.main` 这一个 group 里
- Apple 的 group 机制保证用户**任意时刻只能持有 1 个订阅**——升级（Tune → Studio）/ 降级（Studio → Tune）/ 切换周期（月→年）都是 Apple 自动按比例退款 + 重新计费，零自定义代码
- Group 内 service level 排序：Studio Yearly (4) > Studio Monthly (3) > Tune Yearly (2) > Tune Monthly (1)

### 3.3 试用期 / 退款策略

**推荐**：**Tune / Studio 月订阅各给 7 天免费试用**，年订阅不给试用（年订阅本身已经有 37% off 是钩子）。

- 实施：App Store Connect 配置 Introductory Offer = "Free" / 7 days / 仅 New Subscribers
- 试用 7 天到期前 24h Apple 自动发邮件提醒（用户能取消）
- **不做强制信用卡预扣**——StoreKit 2 试用期是真正"先用后付"，不绑卡也行
- **退款不接管**：用户如果不满意走 Apple 标准退款流程（report a problem），Simone 不自建客服系统

### 3.4 Family Sharing

**推荐启用 Family Sharing**（订阅 + 买断都启用）。

- 理由：氛围音乐是家庭场景产品（开车、居家），家庭分享提升感知价值，对促单转化是正向
- 成本：Apple 不抽额外费用，启用是 App Store Connect 后台勾选项
- 实施：所有 6 个 SKU 启用 `Family Sharing` flag

## 4. 数据指标

### 4.1 核心 3 指标（30 天观察窗口）

| # | 指标 | 目标值 | 工具 |
|---|---|---|---|
| **1** | **Free → Paid 转化率（D7）** | ≥ 3% | App Store Connect → Subscriptions Report |
| **2** | **付费用户中年订阅占比** | ≥ 60% | App Store Connect → Sales and Trends |
| **3** | **试用 → 付费转化率** | ≥ 35% | App Store Connect → Subscriptions Report → "Trial Conversion Rate" |

**为什么是这三个**：
- 指标 1 直接回答"商业化生不生效"。3% 是 freemium 工具类 App 行业基准下沿，能跨过说明定价 / 付费墙触发都对了
- 指标 2 决定 LTV——如果年订阅占比上不去，说明文案 / UI 没把年订阅推成默认。低于 60% 说明 bottom sheet 默认按钮设置有问题
- 指标 3 是"产品价值是否兑现"的体检——试用了 7 天还是不付，说明 Tune 档功能边界定错了（用户用了 7 天还觉得"值不了 $1.99"）

### 4.2 收集工具

| 工具 | 收集什么 | 成本 |
|---|---|---|
| **App Store Connect** | 所有订阅指标、试用转化、留存、ARPU | 0（自带） |
| **Apple App Analytics** | DAU、MAU、Session 时长、留存曲线 | 0（自带） |
| **TestFlight** | v1.3 灰度阶段 50-100 名内测用户口碑反馈 | 0（自带） |
| **不引入第三方埋点** | —— | 保持零服务器、零运维原则 |

**自建埋点**：v1.3 不做。所有数据靠 Apple 后台。理由：
1. 自建埋点要建 server / 收数据 / 看 dashboard，违反"零服务器零运维"原则
2. App Store Connect 的订阅 / 留存数据已经覆盖商业化 3 个核心指标
3. 等 v2.x 真正需要"用户行为漏斗"时再说，v1.3 先确认商业模型本身能跑

### 4.3 "v1.3 商业化成功"判定标准

**全部达成 = 成功**（任一未达成 = 触发 v1.3.1 修补）：
1. v1.3 上架后 30 天累计**付费用户 ≥ 50 人**（任意档位、任意周期）
2. **D7 Free → Paid 转化率 ≥ 3%**（基于 v1.3 上架后 30 天新增用户）
3. **MRR ≥ $100/月**（验证商业模型可滚动，不是"摆设")
4. **没有用户流失数据上的灾难**——v1.3 上架后 30 天 D7 留存不能比 v1.2 同期低 5pp 以上（确认付费墙没把 Free 用户吓跑）

## 5. 风险与缓解

1. **风险：付费墙触发把 Free 用户吓跑，D7 留存断崖**（最严重风险，违反"不破坏沉浸感"）
   **缓解**：付费墙严格只在 Section 2.1 列出的 5 个主动操作触发；TestFlight 灰度 5 天先看 D3 留存对比 v1.2 基线，留存掉超过 3pp 立刻把 Flow 边界扩大（最坏情况：把"点收藏"从付费墙降级为 Free 可用，只锁"创建 / Direct Input"）；v1.3.1 hotfix 路径必须预留

2. **风险：定价过低导致 ARPU 上不去，规模化付费用户也无法 cover Apple Developer 费用之外的运营**
   **缓解**：v1.3 上架后 30 天监测 ARPU；如果 ARPU < $0.5（即付费用户平均月贡献 < 0.5 美金），说明定价偏低，v1.3.1 把 Tune 月订阅从 $1.99 提到 $2.99、买断从 $9.99 提到 $14.99（提价对老用户不影响，仅对新用户生效）；提价对 Simone 这种"陪伴型工具"风险可控，因为产品差异化在视觉 + 氛围而非价格

3. **风险：BYOK 用户在社区/评论区抱怨"自带 Key 还要付费不合理"，伤害品牌**
   **缓解**：在 Settings 页 BYOK 入口下方明确写一行小字 "Bringing your own key removes API rate limits. Tune / Studio unlock product features."（已经在产品文案层面预先化解）；为 BYOK 用户提供专属 50% off 买断折扣（见 Section 2.3 推荐方案），让"你懂技术 + 你支持我"两个身份都被照顾到；如果舆情仍然强烈，备选方案是把 Tune 让给 BYOK（损失可控，因为 BYOK 用户基数本来就小）

## 战略师整体推荐

Simone v1.3 商业化的本质不是"收钱"，是"在不打断聆听的前提下，把懂氛围的用户筛出来"。所以这版设计三个核心抓手——

**Flow 给完整电台体验（播放无限、Evolve 全档、5 个默认 visualizer），只锁"主动选择"和"持有感"**。这个边界是反直觉的（行业惯例是 Free 砍时长砍质量），但符合 Simone "氛围电台"的产品定位。

**付费墙严格只在 5 个主动操作触发，永不在播放中弹出，bottom sheet 形态、Fog 调性、音乐继续放**。这是把"商业化"和"沉浸感"调和的唯一方法。

**早鸟红利做"前 14 天"而非"前 100 名"**，因为零服务器架构下后者无法精确实施，而前者用 StoreKit Introductory Offer 一行配置搞定，用户体感上等价。

定价数字（$1.99 / $4.99 月、$9.99 / $19.99 买断、年订阅 $14.99 / $39.99）沿用 SimonePlan 已有规划，新增的是"年订阅推荐 + 7 天试用 + Family Sharing 全开"——这三个是 freemium iOS App 上架后 ARPU 上得来的关键杠杆。

BYOK 和付费分层独立维度（BYOK 解决配额，付费分层解决功能），但给 BYOK 用户专属 50% off 买断作为温和差异化。
